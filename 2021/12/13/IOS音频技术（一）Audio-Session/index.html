IOS音频技术（一）Audio Session

发表于 2021-12-13
hh

简介

Audio Session是一个用来管理音频的类，你的App通过Audio Session来告诉系统你将如何使用硬件设备，比如麦克风、扬声器。Audio Session的表现就像是一个中介者。

系统提供了AVAudioSession这个类来实现对应的功能：

  • 设置你的App的Audio Session表现行为（设置Category和Mode）

  • 接受通知，比如Audio中断、或者路由改变（如用户插入耳机）

  • 配置音频设备（采样率、通道数等）

这个AVAudioSession是单例的，在App启动之后系统会自动初始化。

配置Audio Session

系统提供了很多种Category，通过设置不同的Category，可以表明你的App的音频表现，比如当你的App播放音频时是否停止其他App的音频、用户打开静音键后你的App是否还能播放。具体看下面这张表：

Category 按下静音键or锁屏的时候静音 中断其他无法混合播放的App Allows audio input (recording) and output (playback)
AVAudioSessionCategoryAmbient Yes No Output only
AVAudioSessionCategorySoloAmbient (Default) Yes Yes Output only
AVAudioSessionCategoryPlayback No Yes by default; no by using override switch Output only
AVAudioSessionCategoryRecord No (锁屏时录音继续) Yes Input only
AVAudioSessionCategoryPlayAndRecord No Yes by default; no by using override switch Input and output
AVAudioSessionCategoryMultiRoute No Yes Input and output

通过setCategory:mode:options:error:这个API来配置。比如你要实现音频可以在后台播放，则使用Category为AVAudioSessionCategoryPlayback，同时在Xcode的Capabilities中开启Background Mode

最后通过setActive:error:来让配置生效。

系统默认配置

系统默认实现的Audio Session配置如下：

  • 支持播放、不支持录音
  • 打开静音键后，App的音频变为静音
  • 锁屏后，App的音频变为静音
  • 当App的音频在播放时，其他App的音频静音

有时我们要播放App时，需要判断其他App是否在使用Audio，这时可通过secondaryAudioShouldBeSilencedHint这个属性来获取，为true时说明在playing
